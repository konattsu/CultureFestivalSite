使用技術等

このサイトの内側についてです。 ゲームなども無く少し専門的な内容のため、閲覧する人を限定する可能性があります。

サーバー

Github Pagesにカスタムドメインを適用しています。ドメインは別サービスで購入しました。当初はサーバーも借りる予定でしたが、Github Pagesの帯域幅が月100GBあり利用規約も大丈夫そうだったので使うことにしました。デプロイも楽なのでこっちにしてよかったです。

ローカルストレージ

本サイトではローカルストレージを使用しています。ハンバーガーメニューを開いていただくといくつかの設定があると思いますが、これを一時的に保持しています。サイズは数十バイトです。無効にしていただいても問題なく使用できます。
似た機能としてcookieが存在しますが、こちらは使用していません。
また、使用者の情報収集は一切していません。(Github Pagesの規約を見て微妙なラインだった、万が一違反すると怖いのでしていない)

Rust

このページをここまで閲覧する人はプログラミングやPCが好きな人以外いないと思うので、最後にRust言語はいいぞというのを概要、強み、弱みの順にしたいと思います。強み、弱みは個人的感想がかなり混入しており、筆者はc,py,rsのみの経験です。

概要

メモリ安全、安全な平行性を目標として作成された言語。c/cpp並みの低次元を操作できる。

強み

- メモリ安全
大体のエラーはビルド段階でエラーとなります。所有権、借用の仕組みでこれらを制御しています。並行性、非同期になってもこれらを十分に活用できます。

- 端的に記述できる
例えばヌルポインタがありません。cではファイルポインタの作成時に失敗するとNULLが代入されますが、RustではResult型を利用します。これ用の"?"演算子が用意されており、エラーの委譲が非常に楽です。いちいち if(fptr == NULL) のような記述が必要ありません。他にも便利な構文が色々あり、開発言語を cpp => rs にすると行数が減るという話をよく聞きます。

- 大体なんでもいける
本当に色々できます。組み込み、wasm、サーバー、CliAppなどです。FirefoxのエンジンはRustで開発されてたりします。Gui(特にGPU)を扱うのが苦手と思い込んでいたのですが、そこまででも無いっぽい感じです。一回も使用していないので何とも言えません...今度試します。

- rustup, cargo crates.io
これらは依存関係、ビルド、パッケージの管理をしてくれます。brew一行で導入でき、操作も簡単です。

- 標準lint clippy
何より標準で使えるのが嬉しいです。設定を変更したいときはルートディレクトリにrustfmt.tomlを置くだけ。また、lintとしての性能もかなり上位にランクインすると思います。

- c/cpp を呼び出せる
既存のc/cppで記述されたコードを呼べます。そのため移行時に少しずつrsに書き換えられます。

弱み

- 少し難しい
有名な話かと思いますが、構文が少し難しいです。ですが、公式の解説ページ で1から詳しく解説してくれているので、他言語の経験があればそこまで苦にはならないと思います。英語を翻訳した内容ですが、自然な日本語で、訳注もあり読みやすかったです。

- 重い
コンパイル時間が長いです。上述のバックエンド用のコードを最適化でコンパイルすると7分ぐらいかかりました。また、外部モジュールをたくさん入れているとvscodeでもclippyなどの読み込みが長くなります。

rustやろうか迷っている人は学習したところで損しないと思います。やりましょう!!
